# Spec: 引用消息 (Quote Message)

## 背景

用户在对话过程中经常需要针对某条消息追问。当前只能在输入框手写上下文，无法明确关联到具体消息，导致 agent 理解偏差。

## 目标

1. 用户可以引用当前 session 内的任意消息
2. 引用内容显示在输入框上方，用户可追加提问
3. 发送后，引用内容作为结构化上下文随 prompt 一起发给 agent

## 功能设计

### 1. 引用触发

每条消息（user / assistant）hover 时，在右上角显示引用按钮：

```
┌─────────────────────────────┐
│ You                    [📎] │  ← hover 时出现引用图标
│ 这个函数的性能有问题        │
└─────────────────────────────┘
```

点击引用按钮后：
- 引用卡片出现在输入框上方
- 输入框获得焦点，用户可输入追问

### 2. 引用卡片 (Quote Card)

输入框上方的引用区域，支持多条引用堆叠：

```
┌─────────────────────────────────────────┐
│ ┌─────────────────────────────────────┐ │
│ │ 📎 You · 2 min ago            [✕]  │ │
│ │ 这个函数的性能有问题               │ │
│ └─────────────────────────────────────┘ │
│ ┌─────────────────────────────────────┐ │
│ │ 📎 Pi · 1 min ago              [✕]  │ │
│ │ 我已经优化了缓存策略...           │ │
│ └─────────────────────────────────────┘ │
├─────────────────────────────────────────┤
│ [输入框]                                 │
└─────────────────────────────────────────┘
```

- 每条引用显示：发送者 + 相对时间 + 内容摘要（max 2 行，超出截断）
- 每条引用右侧有 ✕ 可单独移除
- 引用区域整体有 ✕ 或"清除全部"

### 3. 数据模型

```typescript
interface QuotedMessage {
  messageId: string
  role: 'user' | 'assistant'
  content: string        // 纯文本摘要，max 200 chars
  timestamp: number
}
```

存储在组件 state，不持久化（刷新后消失，合理——引用是即时操作）。

### 4. 发送逻辑

发送时将引用内容作为结构化上下文拼入 prompt：

```typescript
// sendPrompt 时构建最终文本
let finalText = ''
for (const q of quotes) {
  finalText += `[Quoted ${q.role} message]:\n${q.content}\n\n`
}
finalText += userText
```

这样 agent 能清晰区分引用内容和用户的新问题。

### 5. 作用范围：仅当前 Session

**当前 session only**，原因：

| 考量 | 说明 |
|------|------|
| 上下文连贯性 | 跨 session 引用会引入 agent 不知道的上下文，降低回答质量 |
| 实现复杂度 | 跨 session 需要从其他 session 的 JSONL 读消息，增加 IPC 和异步 |
| UX 简洁性 | 引用当前对话更直觉，像 Slack reply to thread |

**未来扩展**：数据模型预留 `sessionPath` 字段，UI 可加 "从其他 session 引用" 入口。

### 6. 引用在 Chat 中的渲染

发送后，用户消息中引用部分渲染为灰色引用块：

```
┌──────────────────────────────────┐
│ You                              │
│ ┌──────────────────────────────┐ │
│ │ 📎 Pi · 1 min ago           │ │  ← 灰色背景，小字
│ │ 我已经优化了缓存策略...     │ │
│ └──────────────────────────────┘ │
│ 缓存命中率提升了多少？           │  ← 用户实际输入
└──────────────────────────────────┘
```

### 7. 选中文本引用（增强）

支持选中消息中的部分文本后引用，而非整条消息：

1. 用户选中消息中的一段文字
2. 出现浮动工具栏：`[引用]`
3. 点击后仅引用选中部分

整条引用和选中文本引用共用同一个 Quote Card 区域。

## 交互流程

```
用户 hover 消息 → 出现 [📎] 按钮
  │
  ▼
点击 [📎] → Quote Card 出现在输入框上方
  │
  ├─ 继续添加其他引用（可叠加多条）
  │
  ├─ 点击 ✕ 移除单条引用
  │
  ├─ 输入框输入追问文字
  │
  ▼
点击 Send → 引用内容 + 追问文字拼接发送
  │
  ▼
Chat 中渲染：引用块（灰色）+ 用户文字（正常）
  │
  ▼
引用清空，输入框恢复初始状态
```

## 边界情况

| 场景 | 处理 |
|------|------|
| 只引用不输入文字 | 允许发送，agent 收到纯引用，自然追问 |
| 引用 streaming 中的消息 | 只能引用已完成的消息，streaming 中不显示引用按钮 |
| 引用 tool_call / tool_result | 只引用 text 类型的 block，忽略其他类型 |
| 引用过长消息 | content 截断 200 chars，发送时用完整内容 |
| 切换 session | 清空引用（引用属于当前 session） |
