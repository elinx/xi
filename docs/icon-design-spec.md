# Xi Icon Design Specification

## 1. 产品定位

Xi（溪）是一个划时代的 AI 编码工具，核心理念：

- **Session as Branch** — 把 session 当作 git branch 一样管理
- **Session 是第一公民** — 当上下文长度趋近无限时，基于单 session 做 compact 的主流设计终将过时，session 的分支与切换才是未来
- **Xi = 溪 = 分支** — 中文名"溪"本身就包含水流分岔的意象，与 branch 天然同构

### 竞品坐标

| 产品 | 理念 | 图标风格 |
|------|------|----------|
| Git | 版本控制 / branch | 红橙菱形分支图 |
| VS Code | 编辑器 | 蓝色简约双色 |
| Cursor | AI 编辑器 | 紫色渐变菱形 |
| Claude | AI 对话 | 橙棕旋涡/火焰 |
| Xi | Session 分支管理 | ? |

Xi 的图标需要传达的不是"编辑器"也不是"AI"，而是**分支·流动·汇聚**。

---

## 2. 设计理念

### 2.1 核心隐喻：溪水分支

```
源头 ●
      \
       ●──── ξ ────●  主 session
      /              \
     ●                ●  分支 session
```

- **溪（stream）**：水流从一个源头出发，遇到地形自然分岔
- **Branch**：每个分岔点就是一个 fork，每条支流就是一个独立 session
- **汇聚与分散**：session 之间可以 fork、可以独立流淌、最终可能 merge 回主干

### 2.2 视觉关键词

| 关键词 | 释义 | 视觉表达 |
|--------|------|----------|
| 分支 | session 的核心操作 | 分叉线条、树状结构 |
| 流动 | 溪水的动态 | 曲线、渐变、方向感 |
| 源头 | 每条溪流的起点 | 单点汇聚 |
| 无限 | 上下文趋近无限 | 开放结构、不封闭 |

### 2.3 品牌性格

- **克制** — 不是炫酷科技风，是深思熟虑的工具感
- **温暖** — 橘色系，溪水的温度感
- **现代** — 简约、几何、无多余装饰
- **专业** — 开发者工具，不是消费级应用

## 3. 视觉规范

### 3.1 画布

- 尺寸：512 × 512 px（SVG 矢量源文件）
- 圆角矩形：`rx=108`（对应 macOS squircle 风格）
- 安全边距：内容区距边缘 ≥ 24px

### 3.2 配色

#### 主色板

| 用途 | 色值 | 预览 |
|------|------|------|
| 背景（暖灰渐变起） | `#292524` | ■ |
| 背景（暖灰渐变止） | `#1c1917` | ■ |
| 主色（橙色） | `#f97316` | ■ |
| 主色深（深橘） | `#c2410c` | ■ |
| 主色浅（亮橘） | `#fb923c` | ■ |

#### 辅助色

| 用途 | 色值 | 说明 |
|------|------|------|
| 微光边框 | `#f97316 @ 35%` | 橙色描边低透明度 |
| 分支线条 | `#f97316 @ 40-60%` | 低于主文字的视觉权重 |
| 节点圆点 | `#f97316 @ 60-70%` | 半透明实心圆 |

### 3.3 字体

- **ξ（小写 Xi）**：衬线体 `Georgia` / `Times New Roman`
  - 尺寸：320–360px（需根据整体构图调整）
  - 垂直位置：y ≈ 230–240（视觉居中，非度量居中，ξ 的视觉重心偏下需手动上移）
  - 字重：Regular / Bold
- ξ 的衬线体形态天然具有三横两竖的分叉结构，本身就是"分支"的视觉符号

### 3.4 几何元素

| 元素 | 规范 | 用途 |
|------|------|------|
| 微光边框 | 2px, `#f97316 @ 35%`, 距外框 12px, `rx=100` | 精致科技感 |
| 分支线 | 2–4px, `stroke-linecap: round`, 曲线用贝塞尔 | 溪水分岔 |
| 节点圆 | r=6–8px, 实心填充 | git 风格的分支节点 |
| 波纹弧 | 1.5–2px, 低透明度, 正弦曲线 | 水流波纹（可选） |
| 源头点 | r=8px, 实心填充, 中等透明度 | session 起点 |

## 4. 图标方案

### 方案 A：纯 ξ（极简基准）

最简洁的形式：暖灰底 + 橙色衬线 ξ + 微光边框。

- 优点：识别性强，缩放友好
- 缺点：没有直接传达"分支"理念
- 适用：需要极致简约的场景

### 方案 B：ξ + 底部分支线

ξ 下方从中心点向下伸出三条分叉线，模拟溪水分流。

- 优点：直接表达"分支"概念
- 缺点：ξ 和分支线在视觉上可能割裂
- 注意：分支线视觉权重需低于 ξ，透明度 40–60%

### 方案 C：ξ + 尾部延伸分支

利用 ξ 字形本身右侧的尾巴笔画，自然延伸出两条支流曲线。最含蓄的分支表达。

- 优点：字形与图形一体化，不割裂
- 缺点：小尺寸下尾巴延伸可能不清晰
- 注意：延伸线需与 ξ 笔画粗细和弧度一致

### 方案 D：纯线条 ξ（几何骨架）

不用字体，用纯线条画出 ξ 的三横两竖骨架结构，底部自然延伸出三叉分支 + 节点。

- 优点：工程感强，与 git 图标精神呼应
- 缺点：丢失了字体的优雅感
- 适用：强调"开发者工具"定位

### 方案 E：源头→ξ→分支（完整生命周期）

顶部一个源头圆点 → 向下流入 ξ → ξ 底部分叉散开为三条支流。表达 session 从创建到分支的完整叙事。

- 优点：叙事最完整，理念传达最清晰
- 缺点：元素最多，小尺寸下可能拥挤
- 注意：源头和分支线需用渐变透明度，避免喧宾夺主

## 5. 多尺寸适配

图标需要在以下尺寸下保持清晰可辨：

| 尺寸 | 用途 | 要求 |
|------|------|------|
| 1024px | macOS App Store | 完整细节 |
| 512px | macOS Dock / Windows 大图标 | 完整细节 |
| 256px | 通用 | 保留分支线和节点 |
| 128px | 文件管理器 | 主元素清晰 |
| 64px | 小图标 | 去掉波纹等装饰细节 |
| 48px | Windows 任务栏 | ξ + 边框 |
| 32px | 网页 favicon | 仅 ξ |
| 16px | 极小图标 | 仅 ξ 轮廓 |

### 缩放降级规则

1. **≤ 64px**：移除波纹弧线
2. **≤ 48px**：移除微光边框
3. **≤ 32px**：移除分支线和节点，仅保留 ξ
4. **≤ 16px**：ξ 字号可适当缩小确保不溢出

## 6. 输出文件

### 源文件

| 文件 | 位置 | 说明 |
|------|------|------|
| SVG | `build/icon/icon.svg` | 矢量源，修改后重新生成 |
| 生成脚本 | `build/icon/generate-icons.mjs` | SVG → PNG/ICO/ICNS |

### 产物

| 文件 | 位置 | 说明 |
|------|------|------|
| PNG 各尺寸 | `build/icon/{16,24,32,48,64,96,128,256,512,1024}.png` | 多尺寸 |
| ICO | `build/icon/icon.ico` | Windows |
| ICNS | `build/icon/icon.icns` | macOS |
| Electron 图标 | `out/main/icon.png` | 运行时（512px） |

### 生成命令

```bash
node build/icon/generate-icons.mjs
```
